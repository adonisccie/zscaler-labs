ZPA Log Streaming

ZPA Log Streaming Service (LSS) exports private access telemetry — user activity, app connector status, and audit events — to any SIEM over TCP/TLS. In this lab you deploy a dedicated Log Streaming Connector, configure a Log Receiver pointing at your SIEM VM, and verify that user activity logs arrive in real time.

Lab 3: ZPA Log Streaming

LAB 3

Scenario: Your security team needs ZPA user-activity events in the SIEM (siem.patraining.safemarch.com). You deploy a dedicated Log Streaming Connector, wire up a Log Receiver, expose the SIEM over ZPA so students can SSH in, and confirm that logs arrive in real time.

Time: ~20 minutes
Service: Zscaler Private Access (ZPA)
💭 THINK FIRST · LOG STREAMING SERVICE

ZPA LSS uses a dedicated App Connector to ship logs from the ZPA cloud to your SIEM. Think through why a separate connector is recommended and what data you actually get before configuring.

  • LSS uses a separate App Connector Group from your application connectors. What operational reason justifies this separation?
    Q1 — MODEL ANSWER
    Log streaming generates a continuous high-throughput TCP stream to the SIEM. If this traffic ran through the same connector as application access, it would compete for bandwidth and connection slots with production app traffic. Separating LSS into its own connector group ensures log delivery doesn't degrade application performance, and lets you scale each independently — more connectors for apps, fewer (but dedicated) for log forwarding.
  • A Log Receiver is configured with a Log Type of "User Activity." What events does this capture, and how is it different from "App Connector Status" logs?
    Q2 — MODEL ANSWER
    User Activity logs capture per-session ZPA access events: who connected, to which App Segment, from which client, at what time, and with what outcome. These are the records you'd use for UEBA, compliance reporting, and incident investigation. App Connector Status logs capture infrastructure health events — connector up/down, tunnel statistics, certificate expiry warnings. They're for ops monitoring, not user behaviour analytics.
  • Why do you need to create a ZPA App Segment and Access Policy for the SIEM itself (siem.patraining.safemarch.com:22)?
    Q3 — MODEL ANSWER
    The SIEM is an internal server on the private network — just like the HVAC app. Without a ZPA App Segment, it is invisible and unreachable from the client VM (which has no direct route to internal subnets). Creating an SSH App Segment on port 22 and an Access Policy exposes the SIEM through ZPA so you can SSH in via ZCC to verify the log stream. This also demonstrates that ZPA can protect admin/SSH access to servers, not just web apps.
Task 3.1 Deploy the Log Streaming Connector ▾
  1. In the Experience Center, navigate to Infrastructure > Private Access > App Connectors > App Connector Groups.
  2. Select + Add App Connector Group and configure:
    • Name: Log Streaming
    • Status: Enabled
    • App Connector Certificate: select the Connector certificate from the dropdown
    • City: enter your location city
  3. Select Save.
  4. In Skytap, open the Corp: Log Streaming Connector VM console.
  5. Run the following command to retrieve the pre-generated provisioning key (OAuth token):
    • cat /etc/issue
  6. Copy the token value displayed.
  7. Back in the Experience Center, navigate to Infrastructure > Private Access > App Connectors > App Connectors.
  8. Select + Add App Connector and configure:
    • App Connector Group: Log Streaming
    • Provisioning Key / OAuth Token: paste the token from the VM
    • Certificate: select the Connector certificate
  9. Select Save and wait for the connector to show a green Connected status.

Task 3.2 Configure the Log Receiver ▾
  1. Navigate to Infrastructure > Private Access > Log Streaming Service > Log Receivers.
  2. Select + Add Log Receiver.
  3. On the Log Receiver tab, enter:
    • Name: Syslog-UserActivity
    • Domain: siem.patraining.safemarch.com
    • TCP Port: 514
    • TLS Encryption: Disabled
    • App Connector Groups: Log Streaming
  4. On the Log Stream tab, configure:
    • Log Type: User Activity
    • Feed Output Type: CSV
  5. Select Save.

Note: The Log Receiver targets the SIEM using its internal hostname. The Log Streaming Connector resolves this hostname and forwards the TCP stream to port 514 on the SIEM. No syslog agent is needed on the SIEM — it's already listening on 514/tcp.

Task 3.3 Create an App Segment and Policy for SSH to SIEM ▾

App Segment — SSH to SIEM

  1. Navigate to Policies > Private Access > App Segments.
  2. Select + Add App Segment and configure:
    • Name: SSH to SIEM
    • Applications: siem.patraining.safemarch.com
    • TCP Ports: 22–22
    • Segment Group: CorpApps
    • Server Group: CorpServers
  3. Select Save.

Access Policy

  1. Navigate to Policies > Private Access > Access Policies.
  2. Select + Add Policy Rule and configure:
    • Rule Name: Allow SSH to SIEM
    • Action: Allow
    • App Segments: SSH to SIEM
  3. Select Save and Activate.

Task 3.4 Verify Log Delivery via SSH ▾
  1. On the Linux Client VM, ensure ZCC is running and connected to your ZPA tenant.
  2. Open a terminal and SSH to the SIEM:
    • ssh student@siem.patraining.safemarch.com
    • Password: Zscaler123!
  3. Once connected, tail the syslog in real time:
    • tail -f /var/log/syslog
  4. In a second terminal, generate ZPA activity by browsing to the HVAC application (from Lab 2) or any other ZPA-proxied resource.
  5. Return to the syslog tail and look for lines containing User Activity zpa-lss — these confirm the Log Streaming Connector is forwarding events.
  6. Press Ctrl+C to stop the tail when verified.

Verify: Each ZPA session appears as a CSV row in the syslog stream. Fields include timestamp, user, app segment, connector, policy decision, and bytes transferred.

MENTAL NOTE
  • Dedicated Connector Group: ZPA LSS requires its own App Connector Group to prevent log throughput from competing with application traffic.
  • Log Receiver: Defines the SIEM destination (hostname + port + protocol); Log Type controls what is captured — User Activity for per-session events, App Connector Status for infrastructure health.
  • TCP Stream: LSS ships logs as a TCP stream (TLS optional) — not via syslog.
  • SIEM Verification: Access the syslog server over ZPA SSH (not direct network connection) to confirm logs are flowing.
