ZPA App Connector & Browser Access

App Connectors are the on-premises agents that connect your internal applications to the Zscaler cloud. Browser Access extends Zero Trust private access to clientless web users — no VPN, no ZCC required. Build both in this lab.

Lab 2: ZPA App Connector & Browser Access

LAB 2

Scenario: Your organisation runs an internal HVAC control web application on <Student FQDN>. You need to make it accessible to HVAC technicians via a browser without a VPN or ZCC, using ZPA Browser Access with a managed certificate.

Time: ~20 minutes
Service: Zscaler Private Access (ZPA)
💭 THINK FIRST · ZPA ARCHITECTURE

App Connectors establish outbound-only tunnels to the Zscaler cloud — no inbound firewall ports needed. Browser Access extends this to users without ZCC. Think through the data path before configuring.

  • An App Connector makes an outbound-only connection to the Zscaler cloud. Why does this eliminate the need to open inbound firewall ports, and what security advantage does this provide over a traditional VPN concentrator?
    Q1 — MODEL ANSWER
    The App Connector initiates the connection to Zscaler's cloud (ZPA Public Service Edge) over TLS on port 443 — a standard outbound port already open in most firewalls. The internal network never needs to accept inbound connections from the internet, eliminating the attack surface that a VPN concentrator exposes. A VPN endpoint is directly internet-reachable and must be patched against exploit chains; the App Connector has no internet-facing socket.
  • Browser Access serves users who cannot run ZCC (e.g. BYOD, third-party contractors). What ZPA component terminates the HTTPS session from the browser and what component forwards the request to the internal app?
    Q2 — MODEL ANSWER
    The ZPA Public Service Edge terminates the browser's HTTPS session using the managed certificate (your external domain, e.g. hvac.zs000149.labtenant.com). After verifying the user's identity via the IdP, it proxies the request through the App Connector tunnel to the internal web app. The App Connector is the last-mile relay — it forwards HTTP to the internal server on port 80 on behalf of the cloud.
  • You assign the App Segment to a Segment Group and a Server Group. What is the architectural purpose of these groupings?
    Q3 — MODEL ANSWER
    A Segment Group logically bundles App Segments for policy reference — Access Policies reference Segment Groups, not individual App Segments, so you can apply a single rule to many apps. A Server Group maps App Segments to the App Connectors that can reach those internal servers. This separation lets you have multiple App Segments use the same connector infrastructure while keeping access policy clean.
Task 2.1 Create an App Connector Group ▾
  1. In the Experience Center, navigate to Infrastructure > Private Access > App Connectors > App Connector Groups.
  2. Select + Add App Connector Group.
  3. Enter the following settings:
    • Name: OnPrem
    • Status: Enabled
    • App Connector Certificate: select the Connector certificate from the dropdown
    • City: enter your location city
  4. Select Save.

Task 2.2 Enrol the App Connector VM ▾
  1. In Skytap, open the Corp: ZPA App Connector VM console.
  2. Run the following command to retrieve the pre-generated provisioning key (OAuth token):
    • cat /etc/issue
  3. Copy the token value displayed.
  4. Back in the Experience Center, navigate to Infrastructure > Private Access > App Connectors > App Connectors.
  5. Select + Add App Connector and enter:
    • App Connector Group: OnPrem
    • Provisioning Key / OAuth Token: paste the token from the VM
    • Certificate: select the Connector certificate
  6. Select Save and wait for the connector to show a green Connected status.

Note: The App Connector uses the provisioning key to authenticate with ZPA and establish its outbound tunnel. Once connected, it appears in the App Connectors list with its public and private IP addresses.

Task 2.3 Create the HVAC App Segment and Access Policy ▾

App Segment — HVAC

  1. Navigate to Policies > Private Access > App Segments.
  2. Select + Add App Segment and configure:
    • Name: HVAC
    • Applications: <Student ID>.hvac.safemarch.com
    • Access Type: Browser Access
    • Certificate: Managed
    • External URL: prefix hvac
    • Internal URL: HTTP, port 80
    • TCP Ports: 80–80
    • Segment Group: CorpApps
    • Server Group: CorpServers
  3. Select Save.

Access Policy

  1. Navigate to Policies > Private Access > Access Policies.
  2. Select + Add Policy Rule and configure:
    • Rule Name: Allow HVAC
    • Action: Allow
    • App Segments: HVAC
    • Client Types: Web Browser
  3. Select Save and Activate.

Task 2.4 Test Browser Access to the HVAC Application ▾
  1. In the Experience Center, navigate to Infrastructure > Private Access > Browser Access and copy your External Domain.
  2. On the Linux Client VM, open a terminal and run:
    • dig <your external domain>
    • Verify DNS resolves to a Zscaler IP.
  3. Open Chrome and paste the External Domain URL into the address bar.
  4. Authenticate using:
    • Username: hvac@<Student FQDN>
    • Password: Admin-123!
  5. Confirm the HVAC Control page loads successfully.

Verify: The HVAC application is accessible via browser without any client software installed, confirming ZPA Browser Access is working end-to-end.

MENTAL NOTE
  • Outbound-Only Connectors: App Connectors make outbound-only TLS connections to ZPA — no inbound firewall ports are needed on the internal network.
  • Browser Access: Uses a Managed Certificate + External Domain so any browser can reach private apps without the ZCC client.
  • Policy Chain: App Segment → Segment Group → Server Group → App Connector Group, with an Access Policy rule tying it all together.
  • Access Type: Must be set to Browser Access (not Private Access) for clientless users — wrong type silently breaks access.
